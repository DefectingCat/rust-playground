到目前位置，我们所有的模块都在同一个文件中。当模块变的更大时，我们可能需要将其移动到单独的文件中，从而使代码更加易读。

只需要简单的修改，就可以将我们需要的模块拆分开来。将 `back_of_house` 拆分到名为 `back_of_house.rs` 的文件中后，在 `main.rs` 中任然需要声明 `mod back_of_house;`，在 `mod back_of_house;` 后使用分号，而不是代码块，这将告诉 Rust 在另一个与模块同名的文件中加载模块的内容。。而在 `back_of_house.rs` 文件中，只需要放之前模块中的内容即可。

```rust
// main.rs
mod back_of_house;

pub use back_of_house::Breakfast;

fn main() {
    let meal = Breakfast::summer(String::from("Rye"));
    meal.check_breakfast();
    println!("I'd like a {} toast, please", meal.toast)
}
```

```rust
// back_of_house.rs
#[derive(Debug)]
pub struct Breakfast {
    pub toast: String,
    fruit: String,
}

impl Breakfast {
    pub fn summer(toast: String) -> Breakfast {
        Breakfast {
            toast,
            fruit: String::from("Apple"),
        }
    }
    pub fn check_breakfast(&self) {
        println!("{:#?}", self);
    }
}
```

## 总结

Rust 提供了将包分成多个 crate，将 crate 分成模块，以及通过指定绝对或相对路径从一个模块引用另一个模块中定义的项的方式。你可以通过使用 `use` 语句将路径引入作用域，这样在多次使用时可以使用更短的路径。模块定义的代码默认是私有的，不过可以选择增加 `pub` 关键字使其定义变为公有。
